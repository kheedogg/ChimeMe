## 핵심 요약 (5줄)

1. 10개 화면 중 2개(main, group-detail)만 1차 완성, 나머지 8개는 overview 스텁 상태로 **전체 기획 완성률 25%** (작성 10개 / 필요 40개 문서).
2. 5,000명 운영 기준 가장 큰 갭은 **시간대(Timezone) 정책 미확정** — 슬롯 매칭의 근간이므로 최우선 결정 필요.
3. 법적 필수 요건(약관/개인정보처리방침/계정 삭제)이 미정이며, Apple/Google 앱 심사 통과를 위해 **P0으로 확정 필요**.
4. 그룹 관리(탈퇴/강퇴/해체/권한), 신고/차단, 초대 상세 흐름 등 **소셜 앱 핵심 정책이 전면 미정** 상태.
5. 개발자에게는 Timezone/삭제파이프라인/채팅아키텍처/FCM/업로드큐 5개 기술 결정을, 테스터에게는 동시업로드/시간경계/재생스트레스/오프라인/삭제엣지 5개 시나리오 검증을 요청.

## 2. 5,000명 사용자 기준 비즈니스 갭 분석

### 2.1 그룹 정책
- ❌ 사용자당 최대 그룹 수 미정 (`decisions.md:73`)
- ❌ 그룹 탈퇴 흐름 미정 (탈퇴자 영상 잔존 여부, 채팅 메시지 처리)
- ❌ 강퇴/추방 정책 미정 (관리자 권한 부여 흐름, 강퇴자 재초대 차단)
- ❌ 그룹 해체 흐름 미정 (소유자 단독 해체 vs 멤버 과반 동의, 잔여 콘텐츠 처리)
- ❌ 멤버 권한 체계 미정 (소유자/관리자/일반 구분 여부)
- ❌ 소유권 이전 미정 (소유자 탈퇴 시 자동 이전 vs 그룹 해체)

### 2.2 신고/차단/안전
- ❌ 사용자 차단/언차단 흐름 전무
- ❌ 콘텐츠 신고 흐름 전무 (영상/메시지/사용자 신고 카테고리)
- ❌ 신고 접수 후 처리(자동 가림 vs 관리자 검토 vs 즉시 삭제) 미정
- ❌ 자동 모더레이션 정책 미정 (음란물 탐지 API, 비속어 필터)
- ⚠️ **Apple 가이드라인 1.2 위반 위험** — UGC 앱은 신고/차단 필수

### 2.3 미성년자 보호
- ❌ 생년월일 수집 정책 미정
- ❌ 만 14세 미만 제한 정책 미정 (KISA 가이드라인)
- ❌ 부모 동의 흐름 미정 (COPPA 글로벌 출시 시)
- ❌ 미성년자 콘텐츠 보호 정책 미정

### 2.4 개인정보/약관/법적 요건 ⚠️ 법적 필수
- ❌ 이용약관 미작성
- ❌ 개인정보처리방침 미작성
- ❌ 약관 동의 위치/시점 미정 (전화번호 입력 화면 vs 온보딩 vs 둘 다)
- ❌ 마케팅 정보 수신 동의 분리 여부 미정
- ❌ 약관 변경 시 재동의 흐름 미정

### 2.5 콘텐츠 보존 정책
- ✅ 영상 7일 확정 (`decisions.md:51`)
- ❌ 7일 후 삭제 파이프라인 미정 (자동 vs 사용자 알림 후)
- ❌ 채팅 메시지 보존 기간 미정 (영구 vs 30일 등)
- ❌ 사용자 본인 영상 다운로드/내보내기 미정

### 2.6 데이터 삭제/계정 탈퇴 ⚠️ 앱 심사 필수
- ❌ 계정 탈퇴 흐름 미정 (Apple 4.5.4 필수)
- ❌ 탈퇴 후 데이터 처리 미정 (즉시 삭제 vs 30일 유예)
- ❌ 탈퇴자가 소유한 그룹 처리 미정
- ❌ 탈퇴자 영상/메시지 잔존 여부 미정
- ❌ GDPR 데이터 내보내기 미정

### 2.7 그룹 초대 흐름
- ❌ 초대 방식 미정 (`decisions.md:72`) — 딥링크/QR/SMS/카카오톡
- ❌ 초대 링크 만료 시간 미정
- ❌ 초대 링크 받은 미설치 사용자 처리 미정 (스토어 유도 → 설치 후 자동 가입)
- ❌ 차단된 사용자가 초대 링크로 진입 시 처리 미정
- ❌ 그룹 최대 인원(8명) 초과 초대 시 처리 미정

### 2.8 시간대(Timezone) 정책 ⚠️ 가장 중대한 갭
- ❌ `hourSlot` 저장 형식 (UTC vs 로컬)
- ❌ 그룹 vs 사용자 시간대 기준
- ❌ DST 처리 (Spring Forward 사라지는 시간, Fall Back 중복 시간)
- ❌ 해외여행 시 timezone 변경 시 슬롯 처리
- ⚠️ `spec/group-detail/data.md:24-26`에 "사용자 로컬" 명시되어 있으나, 다른 timezone 멤버 공존 시 치명적 버그 발생

### 2.9 알림 정책
- ⚠️ 방해 금지 시간 명시(`spec/notifications/overview.md:14`)되어 있으나 기본값 미정
- ❌ 그룹별 알림 ON/OFF 설정 여부 미정
- ❌ 매시 정각 알림 트리거 설계 미정 (`decisions.md:71`)
- ❌ 알림 탭 시 딥링크 동작 미정
- ❌ 시스템 알림 권한 미허용 시 안내 흐름 미정

## 3. 화면별 보완 필요 항목

### 3.1 main
- 누락 시나리오: 그룹 초대 수락 진입 흐름, 최대 그룹 수 도달 시 동작, 추방된 그룹 사라짐 처리
- 누락 상태: 첫 진입(그룹 0개), 네트워크 에러, 그룹 동기화 중
- 정책 결정: 그룹 정렬 기준(최근 활동 vs 수동 핀), 그룹 카드 미리보기 영상 자동 재생 여부

### 3.2 group-detail
- 누락 시나리오: 그룹 설정/관리 진입점, 본인 영상 삭제/교체, 차단된 멤버의 과거 영상 표시
- 누락 상태: 슬롯 168개(7일×24시간) 인디케이터, 본인 미업로드 슬롯 강조, 그룹 해체 직전 경고
- 정책 결정: 슬롯 무한 스크롤 단위, 영상 자동 재생 트리거(스크롤 vs 탭)

### 3.3 group-create
- 누락 시나리오: 초대 링크 공유 채널 선택, 그룹 생성 직후 멤버 0명 상태, 동일 이름 그룹 중복 생성
- 누락 상태: 생성 중 로딩, 네트워크 실패 시 재시도
- 정책 결정: 그룹 이름 규칙(길이/이모지/특수문자), 최소 인원, 비공개/공개 여부

### 3.4 auth-phone
- 누락 시나리오: 약관 동의 체크박스 구성, 베타 화이트리스트 처리, SMS 일일 한도 도달
- 누락 상태: 국가 코드 선택 UI, 잘못된 형식 인라인 검증, App Check 실패
- 정책 결정: 지원 국가 화이트리스트, 약관 동의 분리/통합, 인증 시도 한도

### 3.5 auth-verify
- 누락 시나리오: 코드 재전송 쿨다운, 코드 유효 시간 만료, SMS 미수신 fallback
- 누락 상태: 자동 fill(iOS/Android 차이), 시도 횟수 초과
- 정책 결정: 코드 유효 시간(2분/5분), 재전송 쿨다운(30초/60초), 최대 시도 횟수

### 3.6 onboarding
- 누락 시나리오: 초대 링크 경유 신규 사용자(자동 그룹 가입), 닉네임 중복, 프로필 사진 미설정
- 누락 상태: 생년월일 입력(미성년자 보호), 약관 동의 통합 화면
- 정책 결정: 필수 vs 선택 입력 항목, 미성년자 차단 시점

### 3.7 capture
- 누락 시나리오: 동일 슬롯 영상 교체 정책(횟수 제한), 그룹 멤버 추방된 직후 촬영 진입
- 누락 상태: 권한 거부 안내, 저장 공간 부족, 촬영 중 권한 회수
- 정책 결정: 영상 해상도/코덱(720p H.264 확정 권고), 큐 재시도 상세, 슬롯 마감 직전 경고 UI

### 3.8 chat
- 누락 시나리오: 영상 인용 탭 동작, 추방된 사용자 메시지 표시, 차단 사용자 메시지
- 누락 상태: 메시지 발송 중/실패/재시도, 첨부 영상 로딩
- 정책 결정: 메시지 삭제 시간 제한(5분 등), 페이지네이션 단위(50건 권고), 채팅 보존 기간

### 3.9 profile
- 누락 시나리오: 계정 삭제 유예 기간 진입, 소유 그룹 위임 흐름, 닉네임 변경 제한 도달
- 누락 상태: 다른 사용자 프로필 보기 분리 여부, 차단 사용자 목록
- 정책 결정: 닉네임 변경 주기(7일/30일), 프로필 사진 크롭 규격, 차단 목록 UI 위치

### 3.10 notifications
- 누락 시나리오: 시스템 알림 권한 미허용 시 안내, 그룹별 mute, 추방 후 알림 잔류
- 누락 상태: 토픽 구독 실패, 조용한 시간 활성화 표시
- 정책 결정: 방해금지 기본값(예: 22:00~07:00), 그룹별 설정 지원 여부, 다이제스트 옵션

## 4. 우선순위 결정 요청 사항

### 4.1 개발자(Architect)에게 5개 기술 결정 요청

| # | 결정 항목 | 사유 |
|---|---|---|
| 1 | **시간대 처리 아키텍처** | `hourSlot` UTC vs 로컬 vs 그룹 기준. 데이터 모델/보안 룰/표시 전반에 영향 |
| 2 | **영상 자동 삭제 파이프라인** | Cloud Function vs Storage 수명주기. 5,000 DAU × 7일 = ~210GB × 840K 파일 순환 |
| 3 | **채팅 아키텍처** | Firestore `onSnapshot` 5,000명 동시 리스너 비용/안정성, 외부 SDK 검토 필요 여부 |
| 4 | **FCM 매시간 알림 발송 설계** | Topic vs 개별, 콜드스타트, 방해금지 필터링 위치(서버 vs 클라) |
| 5 | **백그라운드 업로드 큐 안정성** | `expo-task-manager` iOS/Android 차이, 정각 마감 윈도우 처리 |

### 4.2 테스터(Test Engineer)에게 5개 검증 시나리오 요청

| # | 검증 항목 | 사유 |
|---|---|---|
| 1 | **동시 업로드 충돌** | 8명 동시 업로드, `unreadVideoCount` race condition, Firestore 쓰기 충돌 |
| 2 | **시간대 경계 테스트** | 09:59:58 촬영 시작 → 10:00:01 업로드 시 마감 판정, grace 윈도우 정확성 |
| 3 | **영상 재생 스트레스** | 8명 영상 자동 재생, 저사양 기기 디코더 한계, 메모리 leak |
| 4 | **오프라인 → 온라인 전환** | 캐시 표시 → 자동 갱신 → 업로드 큐 재개 정확성 |
| 5 | **계정/그룹 삭제 엣지 케이스** | 그룹 상세 보는 중 그룹 삭제 시 안전 이탈, 권한 잃은 화면에서 액션 시도 |
