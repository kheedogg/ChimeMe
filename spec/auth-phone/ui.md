# auth-phone — UI

## 레이아웃 (위에서 아래로)

### 1. 상단 영역
- 좌측 ← 뒤로가기 (비활성 — 첫 화면)
- 가운데 정렬 로고 영역 (ChimeMe 워드마크, 노란 톤)

### 2. 헤딩 영역
- 큰 굵은 헤딩: **전화번호를 입력해 주세요**
- 서브: "본인 확인 후 ChimeMe를 시작합니다" (옅은 그레이)

### 3. 입력 영역
```
┌──────┬──────────────────────────────┐
│ +82 ▼│ 010-1234-5678                │
└──────┴──────────────────────────────┘
```
- 국가 코드 selector: **+82 고정** (Q-12-11 확정 — 베타 한국 only)
  - 탭 시: "현재 한국 번호만 지원됩니다" 토스트 (selector 비활성 처리)
- 전화번호 입력: 숫자 키패드 자동 활성, 입력 중 `010-1234-5678` 자동 하이픈 포맷
- 입력 길이가 11자리 도달 시 CTA 활성

### 4. 약관 동의 (Q-12-12 확정 — 화면 하단 체크박스)
```
☐  [필수] 이용약관에 동의합니다 (보기 >)
☐  [필수] 개인정보처리방침에 동의합니다 (보기 >)
☐  [선택] 마케팅 정보 수신에 동의합니다
☐  모두 동의합니다
```
- 필수 2개 모두 체크되어야 CTA 활성
- 각 "보기 >" 탭 시 외부 브라우저 또는 모달로 약관 전문 표시 (`spec/legal` 화면)
- "모두 동의" 토글로 일괄 ON/OFF

### 5. CTA 영역 (화면 하단 고정)
- 큰 노란 버튼: **코드 받기**
- 비활성 조건: 전화번호 형식 미충족 OR 필수 약관 미동의
- 활성 시 진한 노랑, 비활성 시 옅은 노랑 + 회색 글씨

### 6. 보조 영역 (CTA 아래)
- 작은 텍스트: "다음 단계에서 받은 SMS 인증번호 6자리를 입력합니다"

## 디자인 토큰
| 토큰 | 값 |
|---|---|
| `color.primary` | `#FFD54F` |
| `color.primaryDisabled` | `#FFE89C` |
| `color.error` | `#E53935` (입력 오류 인라인) |
| `radius.input` | 12 |
| `radius.button` | 999 (pill) 또는 16 |
| `font.heading` | 굵은 sans-serif 28~32px |

## 컴포넌트
- `PhoneInputField` — `{ countryCode, value, onChange, error? }`
- `TermsCheckboxGroup` — `{ required[], optional[], onAllChange }`
- `PrimaryCTA` — `{ label, disabled, loading, onPress }`

## 상태별 UI
- 기본
- 입력 중 (CTA 비활성)
- 모두 충족 (CTA 활성)
- 발송 중 (CTA에 스피너, 입력 잠금)
- 발송 실패 (CTA 활성 복귀 + 인라인 에러)
- rate limit 초과 (CTA 비활성 + 안내 문구 "1시간 후 다시 시도해 주세요")

## 접근성
- 입력 필드 a11y label: "전화번호 입력, +82 한국 번호만 지원"
- 약관 체크박스 a11y label: "필수 동의: 이용약관" 등 명시
- CTA a11y state: 비활성 시 사유 안내 ("필수 약관 동의 필요")
