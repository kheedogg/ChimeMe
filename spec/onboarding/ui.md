# onboarding — UI

## 레이아웃 (1~2 스텝 분리)

### 스텝 1: 생년월일 입력 (Q-12-03)

#### 1.1 헤딩
- 큰 헤딩: **생년월일을 입력해 주세요**
- 서브: "만 14세 이상부터 ChimeMe를 이용할 수 있어요" (옅은 그레이)

#### 1.2 생년월일 입력
- 날짜 picker (휠 형식, iOS/Android 네이티브 picker 사용)
- 기본 표시: 1990-01-01
- 최대 선택 가능: 오늘 - 14년 (실시간 계산)

#### 1.3 CTA
- "다음" 버튼 (만 14세 이상일 때만 활성)
- 만 14세 미만 선택 시:
  - CTA 비활성
  - 인라인 안내: "만 14세 이상부터 가입 가능합니다"

### 스텝 2: 닉네임 + 프로필 사진

#### 2.1 헤딩
- 큰 헤딩: **어떻게 불러드릴까요?**
- 서브: "그룹에서 보일 이름과 사진을 설정해 주세요"

#### 2.2 프로필 사진 영역
```
       ┌──────────┐
       │   👤    │   ← 기본 이모지 아바타 (탭 시 변경 옵션)
       │          │
       └──────────┘
        [사진 변경]
```
- 원형 110×110px
- 미설정 시: 배경 + 첫 글자 이니셜 또는 이모지
- 탭 시 ActionSheet: "카메라로 촬영" / "갤러리에서 선택" / "기본 이미지 사용"

#### 2.3 닉네임 입력
- 라벨: "닉네임"
- 입력 필드: 1~20자 제한
- 헬퍼 텍스트: "1자 이상 20자 이하"
- 글자 수 카운트 우측 표시 (예: `7 / 20`)

#### 2.4 CTA (화면 하단)
- 큰 노란 버튼: **ChimeMe 시작하기**
- 활성 조건: 닉네임 1자 이상

## 디자인 토큰
| 토큰 | 값 |
|---|---|
| `avatar.size` | 110 |
| `avatar.radius` | 999 (원형) |
| `progress.steps` | 2 (상단 점 인디케이터) |

## 컴포넌트
- `StepIndicator` — `{ total, current }`
- `DatePickerField` — `{ value, onChange, maxDate, minDate }`
- `AvatarPicker` — `{ src?, onChange, onSkip }`
- `NicknameInput` — `{ value, onChange, max=20, charCount }`

## 상태별 UI
- 스텝 1 기본
- 스텝 1 만 14세 미만 (CTA 비활성 + 안내)
- 스텝 1 → 2 전환 (좌측 슬라이드 애니메이션)
- 스텝 2 닉네임 입력 중 (글자 수 실시간 업데이트)
- 스텝 2 사진 업로드 중 (아바타 스피너)
- 스텝 2 제출 중 (CTA 스피너 + 입력 잠금)
- 닉네임 길이 초과 (테두리 빨강 + 에러 인라인)

## 빈 상태 / 에러
- 사진 업로드 실패 → 토스트 "사진 업로드에 실패했어요. 다시 시도해 주세요"
- 닉네임 부적절 (욕설/특수문자) → 인라인 에러 "사용할 수 없는 닉네임입니다"

## 접근성
- 스텝 인디케이터 a11y: "2단계 중 1단계, 생년월일 입력"
- 사진 변경 액션시트 a11y: 각 옵션 명시 라벨
- 닉네임 글자 수 a11y: "20자 중 7자 입력됨"

## 차단 정책
- 스텝 1 입력 후 뒤로가기 불가 (인증 완료된 신규 사용자는 onboarding 완료 강제)
- 앱 강제 종료 후 재진입 시 마지막 스텝부터 재개
