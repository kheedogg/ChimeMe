# onboarding — 신규 사용자 온보딩 (프로필 입력)

## 화면 키
`onboarding`

## 진입로
- `auth-verify` 성공 후 사용자가 **신규**일 때 (= `users/{uid}` 문서가 없을 때)
- 기존 사용자는 이 화면을 건너뛰고 바로 `main`

## 목적
- 그룹 안에서 다른 멤버가 본인을 알아볼 수 있도록 **닉네임**(필수) + **프로필 사진**(선택)을 입력받음
- 이용약관 / 개인정보처리방침 동의 흐름이 별도가 아니라면 이 화면에서 함께 처리

## 사용자 시나리오
1. "어떻게 불러드릴까요?" 닉네임 입력
2. (선택) 프로필 사진 등록 — 갤러리 / 카메라
3. (필요 시) 약관 동의 체크박스
4. "시작하기" 탭 → `users/{uid}` 문서 생성 → `main`

## 정책
- 닉네임: 1자 이상, 20자 이하, 공백/특수문자 일부 제한 (구체 룰은 추후)
- 프로필 사진은 건너뛰기 가능, 기본 아바타(이니셜 또는 이모지)로 대체
- 진입 후 이탈/뒤로가기는 차단 (홈으로 못 가도록)
