# auth-verify — UI

## 레이아웃

### 1. 상단
- 좌측 ← 뒤로가기 (전화번호 입력 화면으로 복귀)
- 가운데: 작은 ChimeMe 워드마크 또는 빈 영역

### 2. 헤딩
- 큰 굵은 헤딩: **인증번호 6자리를 입력해 주세요**
- 서브: `+82 010-1234-**78`로 발송됐어요" (마스킹된 번호 표시)

### 3. 6자리 코드 입력 (OTP 박스)
```
┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐
│  │ │  │ │  │ │  │ │  │ │  │
└──┘ └──┘ └──┘ └──┘ └──┘ └──┘
```
- 6개의 박스 (각 56×64px 권장)
- 박스 사이 간격 8px, 3-3 분리 시 4번째 앞에 간격 16px (가독성)
- 입력 시 박스 보더 진해짐 + 노란 underline
- 자동 채움: iOS `textContentType="oneTimeCode"`, Android `autoFillHints="smsOTPCode"`
- 입력 완료 시 자동 검증 트리거 (확인 버튼 탭 불필요)

### 4. 보조 영역
- 남은 시간 카운트다운: "**4:32** 후 만료" (5분 = 300초)
- 재전송 영역:
  - 쿨다운 중: "재전송 가능까지 **0:42**" (회색 텍스트)
  - 쿨다운 종료: "코드 못 받으셨나요? **재전송**" (재전송 텍스트 노란색 + 탭 가능)

### 5. CTA (선택적)
- 자동 검증 트리거되므로 별도 CTA 없음
- 자동 검증 실패 시: 큰 노란 "확인" 버튼 활성 (사용자가 재시도 의지 표시)

### 6. 보안 안내 (CTA 아래)
- 작은 텍스트: "ChimeMe는 인증번호를 절대 묻지 않습니다"

## 디자인 토큰
- `color.otpBoxBorder` — 옅은 회색 (입력 전), 진한 회색 (입력 후)
- `color.otpBoxBorderFocus` — `color.primary`
- `font.otpDigit` — 굵은 sans-serif 24~28px

## 컴포넌트
- `OTPCodeInput` — `{ length=6, value, onChange, onComplete, autoFocus, masked? }`
- `CountdownText` — `{ secondsLeft, format='m:ss' }`
- `ResendCodeButton` — `{ cooldownSeconds, onPress }`

## 상태별 UI
- 기본 (입력 대기)
- 입력 중 (박스 점진 채움)
- 자동 검증 중 (스피너 오버레이)
- 검증 성공 (체크 마크 짧게 표시 후 다음 화면)
- 검증 실패 (박스 전체 빨강 깜빡임 + 에러 메시지 인라인)
- 코드 만료 (전체 박스 회색 + "코드가 만료됐어요. 재전송해 주세요")
- 시도 횟수 초과 (10회 이상 실패 시 잠금 + "잠시 후 다시 시도해 주세요")

## 접근성
