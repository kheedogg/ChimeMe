# blocked-users — UI

## 레이아웃

### 1. 헤더
- 좌측 ← 뒤로가기 (profile로)
- 가운데: **차단한 사용자**

### 2. 안내 영역 (최상단)
```
차단한 사용자의 영상과 메시지는 보이지 않아요.
차단 해제 시 다시 표시됩니다.
```
- 옅은 회색 텍스트, 카드 또는 인라인 안내

### 3. 차단 사용자 리스트
각 항목:
```
┌──────────────────────────────────────┐
│ [아바타]  민지                       │
│           2026-05-12 차단            │
│                              [해제]  │
└──────────────────────────────────────┘
```
- 아바타: 32×32 원형
- 닉네임 (큰 글씨)
- 차단 일시 (작은 회색 텍스트)
- 우측 "해제" 버튼 (작은 노란 또는 회색 버튼)

### 4. 빈 상태
- 차단한 사용자 0명일 때:
  - 가운데 일러스트 (자물쇠 또는 평화 아이콘)
  - 헤딩: "차단한 사용자가 없어요"
  - 서브: "사용자를 차단하면 이 목록에 표시됩니다"

### 5. 차단 해제 확인 다이얼로그
- 헤딩: "[닉네임]의 차단을 해제하시겠어요?"
- 안내: "차단 해제 시 이 사용자의 영상과 메시지가 다시 표시됩니다"
- 버튼: "취소" / "차단 해제" (노란)

### 6. 로딩 상태
- 리스트 자리에 스켈레톤 3장

## 디자인 토큰
| 토큰 | 값 |
|---|---|
| `blockedUsers.itemHeight` | 72 |
| `blockedUsers.avatarSize` | 32 |
| `blockedUsers.unblockButton.bg` | `color.surfaceMuted` |

## 컴포넌트
- `BlockedUserListItem` — `{ user, blockedAt, onUnblock }`
- `UnblockConfirmDialog` — `{ user, onConfirm, onCancel }`
- `EmptyBlockedState` — 빈 상태 일러스트 + 안내

## 상태별 UI
- 로딩 (스켈레톤)
- 데이터 표시 (리스트)
- 빈 상태
- 해제 처리 중 (해당 항목 스피너)
- 해제 완료 (항목 fade-out 애니메이션 후 제거)

## 접근성
- 리스트 항목 a11y: "민지, 2026년 5월 12일 차단됨, 차단 해제 버튼"
- 해제 후 a11y announce: "민지의 차단이 해제되었습니다"
- 빈 상태 a11y: "차단한 사용자가 없습니다"

## 정렬
- 차단 일시 desc (최근 차단된 사용자 상단)
- 향후: 닉네임 가나다순 정렬 옵션
