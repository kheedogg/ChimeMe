# account-delete — UI

## 표시 형태
풀스크린 흐름 (3단계 스택). 각 단계는 별도 라우트로 push/pop 가능.

## 공통 헤더
- 좌측 ← 뒤로가기 (이전 단계 또는 profile)
- 가운데: **계정 삭제** (작은 글씨)
- 상단 stepper: ● ─ ○ ─ ○ (현재 단계 표시)

---

## 스텝 1: 삭제 영향 안내

### 1.1 메인 영역
```
┌────────────────────────────────────┐
│       ⚠️                            │
│                                     │
│    정말 계정을 삭제하시겠어요?      │
│                                     │
│  ChimeMe 계정을 삭제하면:           │
│                                     │
│  • 모든 영상이 영구 삭제돼요         │
│  • 모든 채팅 메시지가 익명화돼요    │
│  • 가입한 모든 그룹에서 나가져요    │
│  • 복구는 불가능해요                │
│                                     │
│  소유한 그룹: N개                    │
│  (다음 단계에서 처리 방법 선택)     │
└────────────────────────────────────┘
```

### 1.2 하단 CTA (2개)
- 좌측: "취소" (회색 배경, 큰 버튼)
- 우측: "다음" (빨간 톤 또는 진한 회색)

---

## 스텝 2: 소유 그룹 처리

### 2.1 헤딩
- 큰 헤딩: **소유한 그룹을 어떻게 할까요?**
- 서브: "[N개] 그룹의 처리 방법을 선택해 주세요"

### 2.2 그룹 리스트
각 그룹 카드:
```
┌──────────────────────────────────────┐
│  😀  가족 (3명)                       │
│                                       │
│  ○ 그룹 해체 (모든 데이터 삭제)       │
│  ○ 다른 멤버에게 권한 위임            │
│      └─ [멤버 선택 ▼]                │
└──────────────────────────────────────┘
```

#### "그룹 해체" 선택 시
- 라디오 ON
- 하단 안내: "모든 영상과 채팅이 영구 삭제됩니다"

#### "권한 위임" 선택 시
- 라디오 ON
- 하단에 멤버 picker 드롭다운 표시
- 멤버 선택 시 미리보기: "민지가 새 소유자가 됩니다"

### 2.3 모든 그룹 처리 완료 시
- 우측 하단 "다음" 활성 (모든 그룹 선택 완료 + 위임 선택 시 멤버 지정 완료)

### 2.4 소유 그룹 0개인 경우
- 이 단계 자동 스킵 → 스텝 3로 바로 이동

---

## 스텝 3: 최종 확인

### 3.1 메인 영역
```
┌─────────────────────────────────────┐
│         🗑️                            │
│                                     │
│    마지막 확인                       │
│                                     │
│  계정과 모든 데이터가 즉시 영구      │
│  삭제됩니다.                         │
│                                     │
│  확인을 위해 아래에                  │
│  '삭제'를 정확히 입력해 주세요       │
│                                     │
│  ┌─────────────────────────────┐   │
│  │ [텍스트 입력]                │   │
│  └─────────────────────────────┘   │
│                                     │
└─────────────────────────────────────┘
```

### 3.2 CTA
- 큰 빨간 버튼: **계정 영구 삭제**
- 비활성 조건: 입력값 ≠ "삭제"

## 처리 중 화면

```
┌─────────────────────────────────────┐
│         [스피너]                     │
│                                     │
│      삭제 진행 중...                 │
│                                     │
│  1/4 ✅ 그룹 정리 완료               │
│  2/4 🔄 영상 삭제 중...              │
│  3/4 ⏳ 메시지 익명화 대기            │
│  4/4 ⏳ 계정 삭제 대기                │
│                                     │
│  취소할 수 없어요. 잠시만 기다려     │
│  주세요.                             │
└─────────────────────────────────────┘
```
- 단계별 진행 상태 (체크/스피너/대기)
- 백버튼 / 스와이프 차단

## 완료 화면
- 큰 체크 아이콘 (회색)
- 헤딩: "계정이 삭제됐어요"
- 서브: "ChimeMe를 이용해 주셔서 감사합니다"
- 자동으로 5초 후 `auth-phone`으로 이동
- 또는 사용자가 "확인" 탭 → 즉시 이동

## 에러 화면
- 일부 실패 시:
  - 체크/X 혼합 표시
  - "일부 데이터 삭제에 실패했어요"
  - 버튼: "재시도" / "고객센터 문의"

## 컴포넌트
- `StepperHeader` — `{ totalSteps, currentStep }`
- `AccountDeleteStep1` — 안내 화면
- `OwnerGroupDecisionList` — `{ groups, decisions, onChange }`
- `OwnerGroupDecisionCard` — `{ group, decision, onChange }`
- `MemberPickerDropdown` — `{ members, value, onChange }`
- `FinalConfirmStep` — 텍스트 입력 확인
- `DeletionProgressView` — 4단계 진행 표시
- `DeletionCompletedView` — 완료 화면
- `DeletionErrorView` — 부분 실패 화면

## 상태별 UI
- 스텝 1 (기본)
- 스텝 2 (그룹 처리 선택 중)
- 스텝 2 (모두 선택 완료, CTA 활성)
- 스텝 3 (입력 대기, CTA 비활성)
- 스텝 3 ("삭제" 입력 완료, CTA 활성)
- 처리 중 (네비게이션 차단)
- 완료
- 부분 실패 (재시도 가능)
