# account-delete — 계정 삭제 흐름

## 화면 키
`account-delete`

## 진입로
- `profile` 화면 "계정 삭제" 메뉴

## 목적
- 사용자가 ChimeMe 계정과 모든 관련 데이터를 영구 삭제할 수 있도록 제공
- Apple 가이드라인 4.5.4 (계정 삭제 기능 의무) 준수
- GDPR Article 17 (잊혀질 권리) 대응

## 사용자 시나리오
1. profile → "계정 삭제" 탭 → 1단계 안내 화면
2. **1단계**: 삭제 시 영향 안내 → "다음" 또는 "취소"
3. **2단계**: 소유 그룹이 있으면 그룹별 처리 선택 (해체 vs 권한 위임)
4. **3단계**: 확인용 "삭제" 텍스트 입력 → 최종 삭제 트리거
5. 처리 완료 → `auth-phone`으로 강제 복귀 + 안내 토스트

## 정책 (Q-12-06, D-12-09)

### 데이터 처리
| 데이터 | 처리 |
|---|---|
| `users/{uid}` 문서 | 완전 삭제 |
| `memberships/{uid}_*` | 완전 삭제 |
| Storage 영상 (`groups/*/{slot}/{uid}.mp4`) | 완전 삭제 |
| `groups/{gid}/messages` 본인 메시지 | 작성자 `deleted_{hash(uid)}` 익명화 (내용 보존) |
| `blocks` 본인 차단 / 본인 피차단 | 완전 삭제 |
| `profiles/{uid}.jpg` Storage | 완전 삭제 |
| Firebase Auth 계정 | 완전 삭제 |
| FCM 토큰 / 토픽 구독 | 완전 해제 |

### 소유 그룹 처리 (사용자 선택)
- **해체**: 모든 영상/채팅/멤버십/초대 일괄 삭제, 멤버에게 FCM 알림
- **권한 위임**: 새 소유자 지정 후 본인은 일반 멤버로 → 탈퇴 처리

### 유예 기간
- **없음** (즉시 삭제) — 베타 정책
- 향후 검토: 30일 유예 (재가입 시 데이터 복원)

## 안전장치
- 3단계 확인 절차로 실수 방지
- "삭제" 텍스트 정확 입력 필수 (3단계)

- 부분 실패 시 재시도 가능 (idempotent)
