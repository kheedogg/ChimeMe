# report — 신고 흐름

## 화면 키
`report`

## 진입로
- `chat` 메시지 길게 누르기 → "신고하기"
- `group-detail` 영상 길게 누르기 → "신고하기"
- `profile` (타인의) → "사용자 신고"
- `group-settings` 멤버 리스트에서 → "신고"

## 목적
- 부적절한 영상 / 채팅 메시지 / 사용자를 운영자에게 알리기 위한 단일 신고 흐름
- Apple 가이드라인 1.2 (UGC 신고 기능 필수) 준수

## 사용자 시나리오
1. 대상 콘텐츠(영상/메시지/사용자) 길게 누르기 → "신고하기" 선택
2. 신고 사유 선택 (모달 / 바텀시트)
3. (선택) 부연 설명 입력
4. "신고하기" 제출
5. 즉시 신고자 화면에서 해당 콘텐츠 가림 (Q-12-04)
6. 완료 토스트: "신고가 접수됐어요. 검토 후 조치됩니다"

## 정책 (Q-12-04 확정)
- **신고 즉시 신고자 화면에서만 가림** — 다른 멤버에게는 영향 없음
- **동일 콘텐츠 누적 3건 신고 시 그룹 전체 가림** + 관리자 검토 큐 등록
- **허위 신고 5건 누적 시 신고 기능 24시간 정지**
- 신고는 익명 처리 — 피신고자는 신고자가 누구인지 알 수 없음

## 신고 사유 카테고리
- 욕설 / 혐오 표현
- 음란물 / 부적절한 콘텐츠
- 스팸 / 광고
- 폭력 / 위협
- 미성년자 보호 위반
- 기타 (자유 입력)

## 신고 대상 유형
| 유형 | targetType | targetId 형식 |
|---|---|---|
| 영상 | `video` | `{groupId}/posts/{postId}` |
| 메시지 | `message` | `{groupId}/messages/{msgId}` |
| 사용자 | `user` | `{uid}` |

## 미정 / 향후 검토
- 자동 모더레이션 (Cloud Vision API 영상 검사) — 베타 운영 데이터 기반 결정
- 신고 후 사용자에게 진행 상황 알림 여부 (관리자가 조치 완료 시 토스트)
- 운영자 도구 (관리자 대시보드) — Phase 후속
