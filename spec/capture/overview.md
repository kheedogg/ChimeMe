# capture — 3초 영상 촬영

## 화면 키
`capture`

## 진입로
- 그룹 상세에서 본인 슬롯(현재 시간대) 미업로드 시 "지금 촬영" CTA
- (향후) 푸시 알림의 deeplink

## 목적
- 매시간 단위 슬롯에 본인의 3초 영상을 업로드

## 사용자 시나리오
1. 카메라 권한 확인 (없으면 권한 안내 화면으로)
2. 전면/후면 카메라 토글
3. 셔터 영역을 **한 번 탭** → 3-2-1 카운트다운 → 자동 3초 녹화
4. 3초 자동 종료 → 미리보기 → "올리기" 또는 "다시 찍기"
5. 업로드 큐에 등록 → 그룹 상세로 즉시 복귀(백그라운드 업로드 진행)

## 정책
- **업로드 윈도우: 정각 이후 60분(다음 정각 직전까지)** (확정 2026-05-11)
  - 예: 09:00 슬롯은 09:00:00 ~ 09:59:59 사이에만 업로드 가능
  - 다음 정각 도달 시점에 해당 슬롯은 마감, 이후 소급 업로드 금지
- **녹화 트리거: 한 번 탭 → 3-2-1 카운트다운 후 자동 3초 녹화** (확정)
- **재촬영: 무제한** — 업로드 확정 전까지 "다시 찍기" 무한 허용
- **업로드 방식: 백그라운드 업로드 큐** (확정)
  - 큐 등록 직후 화면 복귀, 앱 종료/네트워크 단절 시에도 재개
  - 큐 항목별 상태: pending / uploading / failed / done
  - 슬롯 마감(다음 정각)까지 업로드 완료되지 않으면 큐에서 제거 + 사용자에게 알림
  - 구현: `expo-task-manager` + 재시도 정책
- **카메라 권한 거부 처리**: 설명 화면 + "설정 열기" 버튼으로 시스템 설정 유도

## 인코딩 표준 (D-12-03, D-12-11 확정)
- 코덱: **H.264 (AVC)** — `recordAsync({ codec: 'avc1' })`로 강제
- 해상도: **720p (1280×720)**
- 프레임레이트: **30fps**
- 비트레이트: **2~3 Mbps**
- 컨테이너: **MP4** (`moov` atom 선행)
- 클립 크기: 약 **1 MB / 3초**

## 큐 재시도 정책 (D-12-07 확정)
- 큐 상태 enum: `pending` / `uploading` / `success` / `failed` / `expired`
- 지수 백오프: 1s → 2s → 4s → 8s → 16s, **최대 5회**
- 슬롯 마감 + grace 30초 (D-12-02) 초과 시 자동 `expired` 처리
- Storage 경로 멱등성: `{groupId}/{slotKey}/{uid}.mp4` 고정 (재시도 시 덮어쓰기)
- Firestore 문서: `setDoc(deterministicId)` upsert로 중복 방지

## 슬롯 시간 (D-12-01, D-12-02 확정)
- 슬롯 키: UTC 기반 `YYYYMMDDHH` 저장, 표시만 로컬
- 업로드 허용 윈도우: 정각 ~ 다음 정각 + **grace 30초**
- Security Rule에서 `request.time` 기준 검증
